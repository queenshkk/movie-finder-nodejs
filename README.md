# Projet de fin d'année Programmation Web Backend

## **Technologies spécifiques**

- **Node.js et Express.js :** Utiliser Express.js pour créer l'API back-end.
- **JavaScript Vanilla :** Utilisation de JavaScript sans frameworks

## **Structure du projet**

- **Architecture client-serveur :** Séparation claire entre le front-end (interface utilisateur) et le back-end (API et logique métier).
- **Travail en binôme**

## **Contraintes techniques**

- **Base de données ou fichiers JSON :** Les données peuvent être stockées dans une base de données (comme MongoDB ou PostgreSQL) ou dans des fichiers JSON pour des projets plus simples.
- **Front-end :** Utiliser HTML, CSS, et JavaScript avec un accent sur une bonne UX/UI (éventuellement : responsivité).
- **CRUD :** Implémenter les opérations CRUD sur les données.
- **Au choix**
    - **API externe :** Intégrer au moins une API publique pour enrichir les fonctionnalités (ex : météo, actualités, géolocalisation).
        - ET/OU
    - **Package NPM** : intégrer au moins un package NPM tiers (pas DB, express.js ou tout package basique pour l'usage courant)

## Au moins 1 Fonctionnalité supplémentaire

- **Authentification :** Implémenter une authentification pour sécuriser les accès si le projet s'y prête [utilisation d’outils d’authentification, pas juste un mot de passe en DB].
- **Gestion de fichiers :** Téléchargement, prévisualisation, ou organisation des fichiers côté utilisateur.
- **Progressive Web App (PWA) :** Ajouter des fonctionnalités comme le mode hors-ligne, les notifications push, et l'installation en tant qu'application.
- **Responsive :** L'interface doit être responsive, s'adaptant à différentes tailles d'écran, y compris les mobiles.
- …

## Bonnes pratiques

- **Documentation du code :** JSDoc + intra code
- **Code propre :** linter
- **Documentation technique :** Fournir un fichier expliquant l'installation, l'utilisation, et la structure du projet.
- **Tests fonctionnels :** Proposer des scénarios de test pour valider les fonctionnalités implémentées.
- Respecter une architecture modulaire (ex : séparation des routes, controllers, modèles…)
- Organisation des fichiers (ex : routes/, controllers/, models/, public/, etc.)
- Adopter des conventions de nommage claires (camelCase, PascalCase pour les classes, etc.)
- **Utilisation de git & github :**
    - Un dépôt GitHub commun pour le binôme
    - Commits clairs et significatifs
